		+ Bind Mount: 
			- Map host directory to container (shared). 
			- Syntax: ./host_dir:/container_dir/ 
			- Allows us to work on files in VS Code (instead of container's vim). 
		+ Named Volume: 
			- Managed by Docker. 
			- Syntax: volume-name:/var/lib/postgresql/data/ 
			- Survives container deletion. 
			- Host location (Debian): /var/lib/docker/volumes/ 

	7.- | 
		+ $ docker compose logs -f 
		+ Or just do `up` without `--detach` to see all logs. 

	8.- | 
		+ ENTRYPOINT: 
			- Main command that always runs. 
			- Use when command should always be the same. 
			- Harder to override (with --entrypoint). 
		+ CMD: 
			- Default arguments or fallback command. 
			- Override by simply adding args to `docker run`. 
			- Use for flexibility. 
		+ Best practice: BOTH. Example: 
			ENTRYPOINT ["java", "-jar"]
			CMD ["app.jar"]
			# $ docker run app 			-> java -jar app.jar 
			# $ docker run app another.jar 	-> java -jar another.jar
